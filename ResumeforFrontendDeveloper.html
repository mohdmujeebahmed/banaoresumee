01 · what recruiters expect

Before they read a single bullet, they're scanning for proof.

A recruiter spends 20–30 seconds on first pass. A hiring engineer spends longer, but only if the first pass survives. Here's what both are actually looking for, in order.

recruiter — checklist.sh
$run screen.sh --role="frontend developer"
✓A live portfolio link or deployed project — not just a GitHub profile // dead repos count against you
✓One primary framework done deeply // React, Vue, Angular — pick a lane, don't list all three as "expert"
✓Numbers attached to impact // load time, conversion, bundle size, users — not just "worked on"
✓Evidence of collaboration // PR reviews, design handoff from Figma, cross-team shipping
✓Awareness of performance and accessibility // Core Web Vitals, WCAG — separates junior from mid
✓A resume that parses cleanly through an ATS // no tables, no icons-as-text, real PDF text layer
$6/6 checks passed → moved to phone screen

04 · common mistakes

The bugs in your resume that never make it to a stack trace.

No error message. No red squiggly line. Just silence — and a rejection email three weeks later.

Uncaught

Listing every framework you've ever opened once

Claiming "expert" in React, Vue, Angular, and Svelte reads as expert in none. Pick the one you actually shipped production code in and go deep.

→ prune to 1 primary + 1 secondary
404

Portfolio link that's dead, half-built, or just a GitHub profile

Recruiters click. If it 404s, spins forever, or shows three forked repos with no README, that's the whole impression formed.

→ one deployed, working project
TypeError

Responsibilities instead of results

"Worked on the homepage" tells them nothing. "Reduced homepage load time by 40%" tells them you understand impact.

→ every bullet needs a number
Warning

Design exported to PDF with images or icons as text

ATS software can't read text baked into an image. Your skills section becomes invisible to the very system scanning it.

→ export with a real text layer
Deprecated

Generic objective statements

"Seeking a challenging position to utilize my skills" is filler a recruiter has read a thousand times. Cut it — let your work speak.

→ replace with a 2-line summary
